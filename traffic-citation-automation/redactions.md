# Redactions & Sanitization

This document explains what information was changed or removed from the original project to create this public case study.

---

## Client & Case Data

**Removed:**
- Actual client names and contact information
- Real ticket numbers and case details
- Specific court dates and case outcomes
- Photos of actual traffic citations
- Payment amounts and financial transactions
- Attorney-client communications

**Sanitized to:**
- Generic descriptions of ticket processing
- Percentage-based metrics without volumes
- Feature descriptions without usage data
- Conceptual workflows without real cases

**Why:** Protects client privacy and attorney-client privilege.

---

## Business & Financial Data

**Removed:**
- Exact pricing (consultation fees, retainers, subscription rates)
- Revenue numbers and growth metrics
- Client acquisition costs
- Conversion rates (lead → consultation → retention)
- Specific payment volumes
- Attorney compensation structure

**Changed to:**
- Generic payment workflow descriptions
- Feature availability without pricing
- Relative improvements (e.g., "90% time reduction")
- General business model description

**Why:** Protects business-sensitive pricing and financial information.

---

## Azure & Infrastructure Details

**Removed:**
- Azure subscription IDs and resource names
- Form Recognizer model IDs and endpoints
- Blob storage account names and connection strings
- API keys and secrets
- Exact API usage costs
- Resource group configurations

**Generalized:**
- "Azure Form Recognizer" without deployment specifics
- "Azure Blob Storage" without account details
- "OCR processing" without API endpoint information
- "Document storage" without container names

**Why:** Prevents unauthorized access and infrastructure reconnaissance.

---
